02 — What's Already Running

The integration layer, today.

StrydeOS runs alongside clinical tooling like Physitrack with a clinic admin in the middle, copy-pasting. Every prescription is a workflow tax, every adherence read is a manual lookup. The data exists, the connection doesn't. Closing that gap is the smallest unit of work that unlocks the largest behavioural change in the practice.

The loop today

A person in the middle, data dropped at every step.

Manual and lossy. Every clinic running both tools does this by hand. Programme links get pasted into notes, adherence gets eyeballed in a separate tab, and the loop never closes back to the clinical record. Most physios don't even check.

i. The data you own

The between-visit record

The PMS captures 30 minute snapshots, fortnightly. Physitrack covers the 13 days in between. With API access, both land on the same screen, and adherence stops being a side panel. The single highest-leverage view a clinician can have at handover, discharge, or insurer review.

ii. The half page report

Adherence with PMS context

Adherence in isolation is a number. Read alongside the clinical record, it's a signal: "60% completion in the two weeks following a discharge note flagging mild flare-up." Same data. Different page. One reads as a vanity metric, the other reads as a clinical alert.

Every clinic running both tools is already paying the integration tax. We're proposing to refund it.

03 — The Ask

What we need from partners.

For each clinical tool we integrate with, we need sanctioned access to two endpoints — a write endpoint to assign programmes on clinical events, and a read endpoint to pull adherence back into the dashboard. Both routed on a per-clinic OAuth basis. Engineering, hosting, maintenance, API usage: ours. Partners provide the green light and a sandbox key.

We carry the build. Partners carry the access.
