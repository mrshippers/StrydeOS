The governance bundle

Product hazard log (Ava AI risk register)

Hazard log H1–H6: missed call, wrong-clinic data leak, stale KPI, automation misfire, PHI leak via SDK telemetry, audit-trail loss. Each with mitigations and a quarterly review cadence. Engineering rigor for an AI that routes patient calls — not an NHS submission. docs/CLINICAL_SAFETY_CASE.md

Adverse-event SOP — built around the 72hr ICO clock

4-quadrant triage matrix with named SLAs, notification chain, Firestore record shape, post-incident review template, retention. Wires directly into UK GDPR Art. 33–34 breach reporting. docs/ADVERSE_EVENTS.md + typed contracts in types/adverse-events.ts.

UK data-residency lock — at the runtime layer

All Vercel serverless + cron handlers pinned to lhr1 (London). Closes the international-transfer question (UK GDPR Art. 44–49) at the deploy boundary. Documented in RUNBOOK.md.

Sentry region enforcement — at the SDK layer

Both server and edge configs parse the DSN host at init and refuse anything that isn't ingest.de.sentry.io or ingest.eu.sentry.io. Misconfig = Sentry off, app stays up, [CRITICAL] log surfaces in Vercel.

Sub-processor register (Art. 28 evidence)

Heidi flagged with a hard 2026-06-01 deadline blocking new-clinic enablement. Anthropic added as action required (DPA + zero-retention + input-field lock). Real deadlines, not vibes — and the exact artefact a procurement team will ask to see.
