Clinical safety case (DCB0129/0160-style)

Hazard log H1–H6: missed call, wrong-clinic data leak, stale KPI, automation misfire, PHI leak via SDK telemetry, audit-trail loss. Each with mitigation cross-refs and a quarterly review cadence. docs/CLINICAL_SAFETY_CASE.md

Adverse-event SOP

4-quadrant triage matrix with named SLAs, notification chain, Firestore record shape, post-incident review template, 7-year retention. docs/ADVERSE_EVENTS.md + typed contracts in types/adverse-events.ts.

UK data-residency lock — at the runtime layer

All Vercel serverless + cron handlers pinned to lhr1 (London). No accidental US execution path for PHI. Documented in RUNBOOK.md.

Sentry region enforcement — at the SDK layer

Both server and edge configs parse the DSN host at init and refuse anything that isn't ingest.de.sentry.io or ingest.eu.sentry.io. Misconfig = Sentry off, app stays up, [CRITICAL] log surfaces in Vercel.

Sub-processor register, kept honest

Heidi flagged with a hard 2026-06-01 deadline blocking new-clinic enablement. Anthropic added as action required (DPA + zero-retention + input-field lock). Real deadlines, not vibes.
